# Beeceptor Express Server & Playwright Verification Assignment

This repository contains a basic Express.js server and a Playwright test script to verify endpoint proxy routing using Beeceptor and ngrok.

## Project Structure

```text
├── server/            # Express.js Server
│   ├── server.js      # Main server logic with request/response logging
│   └── package.json   # Server dependencies
└── scripts/           # Playwright Test Suite
    ├── health.spec.js # Test script for synchronous and asynchronous endpoints
    └── package.json   # Test dependencies
```

---

## Step 1: Start the Local Express Server

Navigate to the `server/` directory, install dependencies, and start the server:

```bash
cd server
npm install
npm start
```

The server will start running locally on `http://localhost:3000`. It includes a logging middleware that prints all incoming requests and outgoing responses (including status codes and response times).

---

## Step 2: Expose the Local Server to the Public Internet using ngrok

Since Beeceptor needs to forward requests to your local server, you need to expose it using a tunneling tool like **ngrok**:

1. **Install ngrok** (if you haven't already):
   ```bash
   npm install -g ngrok
   ```
2. **Start the HTTP tunnel** pointing to your server's port (`3000`):
   ```bash
   ngrok http 3000
   ```
3. Copy the public **HTTPS URL** generated by ngrok (e.g., `https://xxxx-xx-xx-xx.ngrok-free.app`).

---

## Step 3: Configure Beeceptor Proxy Rule

Configure your Beeceptor endpoints using the official [Beeceptor Proxy Rule / HTTP Callout Guide](https://beeceptor.com/docs/proxy-rule-http-callout/).

1. Create a new endpoint/mock server on Beeceptor (e.g., `https://your-subdomain.free.beeceptor.com`).
2. Add the following rules on Beeceptor:
   - **`/synchronous` (Proxy Rule)**:
     - Set up a routing rule that forwards all traffic hitting `/synchronous` on Beeceptor to your ngrok public HTTPS URL `/health` (e.g., `https://xxxx-xx-xx-xx.ngrok-free.app/health`).
   - **`/asynchronous` (Mock Rule)**:
     - Create a mock response for `/asynchronous` returning:
       ```json
       {
         "message": "This is an instant response from beeceptor asynchronous callout"
       }
       ```

---

## Step 4: Run Playwright Verification Tests

Once the server is running and the Beeceptor proxy is configured:

1. Update the `baseURL` inside [scripts/playwright.config.js](file:///Users/praneeth/Assignment/beeceptor-assignment/scripts/playwright.config.js) to match your Beeceptor endpoint (e.g. `https://your-subdomain.free.beeceptor.com`).
2. Navigate to the `scripts/` directory, install dependencies, and run the test suite:
   ```bash
   cd scripts
   npm install
   npm test
   ```

This will run the Playwright test verifying both the `/synchronous` and `/asynchronous` endpoints.
